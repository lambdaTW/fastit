## 目的

當今，對話式人工智慧當道，本系列希望使用對話方式，帶入大家進入日常開發的情境，我們深入探討一個充滿活力的程式開發團隊，由年輕有熱情的小獅子和資深充滿智慧的老獅子所組成。這個團隊正積極參與一個新專案的開發，該專案具有重要的程式設計挑戰和目標。

在專案的早期階段，他們討論了專案的需求，並制定了清晰的計劃。他們開始設計專案的架構，並著手實現核心功能。在整個過程中，老獅子提供了寶貴的建議，並確保程式碼的品質和穩定性。

團隊強調了測試的重要性，並將其納入開發流程中。他們積極進行代碼測試，確保專案的可靠性和可維護性。


## 角色介紹
- 小獅 (小獅子)：
```
角色介紹：小獅子是文章中的主要角色之一，是一位新手程式設計師，擁有一些基本的開發知識，但仍在學習中。
內容提要：小獅子在文章中表達對專案目錄的創建、虛擬環境的使用、套件管理、程式編寫規範、使用者驗證等主題的興趣。他提出問題並向老獅請教，逐漸學習如何進行軟體開發的不同方面。
```
- 老獅 (老獅子)：
```
角色介紹：老獅子是文章中的資深程式設計師，具有豐富的開發經驗，是小獅子的導師和指導者。
內容提要：老獅子在文章中充當了導師的角色，向小獅子解答問題，提供建議，教導程式設計技巧，以及如何溝通、由需求端殺死功能，以幫助小獅子逐漸成長為一位更有經驗的開發者。
文章主要涵蓋了軟體開發過程中的不同方面，包括專案目錄的建立、虛擬環境的設置、套件管理、編程規範、使用者驗證、軟體測試等。
```


本系列會帶入 `FastAPI` 做開發主軸，通過小獅子和老獅子的對話，讀者可以了解如何在實際專案中處理這些問題，以及導師與學徒之間的知識傳承過程。
